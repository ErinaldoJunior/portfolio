Overview

The Problem

Service-based businesses — barbershops, clinics, beauty salons — all share the same operational headache: managing appointments, clients, payments, and staff schedules across disconnected tools. Most existing solutions are either too rigid, too expensive, or don't handle the full workflow from booking to payment.

AgendaX was built to solve that. It's a single platform where business owners manage their entire scheduling operation, staff handle their daily appointments, and clients book services through a public-facing page — all under one roof. The system supports multiple businesses on a shared infrastructure, with strict tenant isolation so each company's data stays private.

Architecture

Tech Stack

Next.js 15 + React 19

App Router architecture for server components, streaming, and optimal performance. React 19 for the latest concurrent features.

TypeScript

Full type safety across the entire codebase — from API routes to frontend components. Catches bugs before they ship.

PostgreSQL + Prisma

35 models with complex relations. Prisma ORM for type-safe database access and painless migrations across 117 API endpoints.

Stripe

Subscription billing with trial periods, payment processing for appointments, and webhook handling for real-time payment events.

Resend + Infobip

Dual notification stack: transactional emails through Resend for confirmations and receipts, SMS via Infobip for appointment reminders and campaigns.

Tailwind CSS 4

Utility-first styling for rapid UI development. Custom design system built on top for consistent spacing, colors, and component patterns.

Features

What It Does

01

Interactive Calendar Scheduling

FullCalendar integration with custom availability logic, drag-and-drop rescheduling, and real-time conflict detection. Staff can manage their day visually, and clients see only available slots.

02

Multi-Tenant Architecture

Each business operates in its own isolated environment within a shared database. Row-level security and middleware-enforced tenant isolation ensure complete data separation without the cost of separate databases.

03

Role-Based Dashboards

Four distinct interfaces — admin, business owner, staff, and customer — each tailored to what that user needs. The admin sees platform metrics, the owner sees revenue and team performance, staff see their schedule, and customers see their bookings.

04

Payment & Subscription Engine

Stripe handles subscription plans with free trial periods, per-appointment payments supporting credit card, debit, PIX, and Multibanco. Webhooks keep everything in sync.

05

SMS & Email Notifications

Automated appointment reminders, confirmation emails, and a campaign system that lets business owners send bulk SMS to their client base. Balance tracking prevents overspending.

06

Public Booking Websites

Each business gets a customizable public page where clients can browse services, pick a time slot, and book — no account required. Owners control the branding, colors, and content.

Deep Dive

Challenges & Solutions

Tenant Isolation at Scale

The core challenge of any multi-tenant system is ensuring Company A never sees Company B's data. Rather than using separate databases (expensive and hard to maintain), I implemented row-level filtering through Prisma middleware. Every query is automatically scoped to the authenticated tenant's ID. JWT tokens carry the company context, and middleware validates it before any data access. This gives the isolation of separate databases with the simplicity of a shared schema.

Subscription Billing Logic

Stripe subscriptions sound simple until you deal with trial expirations, failed payments, plan downgrades, and feature limit enforcement. I built a plan-limits system that maps each subscription tier to specific feature caps (number of staff, SMS credits, client capacity). A cron job checks for expired trials daily and automatically restricts access. Stripe webhooks handle payment state changes in real time, so the system always reflects the current billing status.

Reliable Notification Delivery

Appointment reminders are time-sensitive — a missed SMS means a no-show. I implemented a dual-channel approach: email through Resend for detailed confirmations, and SMS through Infobip for time-critical reminders. A cron job runs daily to queue upcoming appointment reminders. Each send is logged and tracked, and the system falls back gracefully if one channel fails. SMS balance is tracked per-company to prevent unexpected charges.

Result

Live in Production

AgendaX is deployed on Vercel with a Neon PostgreSQL database, handling real business operations. The entire platform — 117 API endpoints, 35 database models, four role-based dashboards, Stripe billing, SMS campaigns, and public booking pages — was built and shipped by one developer.
