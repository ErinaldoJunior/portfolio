Overview

The Problem

Recruitment at scale is messy. Recruiters spend hours manually scanning resumes, cross-referencing skills against job requirements, and making gut-call decisions about which candidates to contact. The process is slow, inconsistent, and misses good matches buried in large candidate pools.

Career Connect uses AI to flip that process. Instead of recruiters searching for candidates, the system generates ranked matches based on skills overlap, experience relevance, education fit, and other criteria. Recruiters review the AI's suggestions, accept or reject them, and rate accuracy — feeding data back into the system to improve future results.

Architecture

Tech Stack

Next.js 14 + React 18

App Router with static export for optimized delivery. Server-side rendering where needed, client components for interactive features like the match generation UI.

TypeScript

Full type coverage across the codebase — GraphQL queries, Redux state, component props, and API responses all typed end-to-end.

GraphQL + Apollo Client

The data layer runs on GraphQL. Apollo Client handles caching, CSRF-protected queries, optimistic UI updates, and real-time polling for match generation status.

Redux Toolkit

Global state management for user sessions, table configurations, and filter persistence. Keeps the complex filtering and column customization logic predictable.

Radix UI + Tailwind CSS

45+ headless Radix UI primitives styled with Tailwind. Accessible by default — dialogs, popovers, selects, tooltips — all keyboard-navigable and screen-reader friendly.

TanStack Table + Tiptap

Data-heavy views use TanStack React Table for sorting, filtering, pagination, and column customization. Rich notes use Tiptap with mentions and formatting.

Core Feature

How AI Matching Works

01

Trigger Generation

A recruiter selects a job or candidate and clicks "Generate Matches." The system queues an AI task and begins polling every 2 seconds. There's an 8-hour cooldown per entity and a cap of 5 concurrent generations to manage resource usage.

02

AI Scoring & Analysis

The backend AI analyzes hard skills, work experience, education, seniority, location preferences, and languages. Each match gets an overall relevance score (0-100), a list of matching vs. missing skills, and a requirements match percentage.

03

Results & Relevance Breakdown

Matches appear in real time as generation completes. Each result includes an AI-generated explanation of why this candidate fits — with HTML-formatted reasoning that highlights specific experience, skills overlap, and criteria weights.

04

Decision & Feedback Loop

Recruiters accept, reject, or rate each match. Thumbs-down ratings include structured reasons and custom notes. This feedback trains the AI to improve future match quality — a closed loop between human judgment and machine learning.

Features

What I Built

01

Advanced Data Tables

Candidates, jobs, and matches are displayed in highly configurable data tables powered by TanStack React Table. Users can sort, filter with boolean operators (AND/OR), toggle column visibility, reorder columns, and save custom filter presets for reuse.

02

Quick View Panels

A slide-in panel lets recruiters inspect candidate profiles, job requirements, and match details without navigating away from the table. Match generation can be triggered directly from the quick view — keeping the workflow tight.

03

Rich Text Notes

Recruiters can attach notes to candidates, jobs, and matches using a Tiptap-powered editor with formatting, mentions, and placeholders. Notes persist across sessions and are tied to specific entities.

04

Observability

Grafana Faro SDK is integrated for real user monitoring — tracking performance, errors, and user interactions. This gives the team visibility into frontend health without relying solely on backend logs.

Deep Dive

Challenges & Solutions

Real-Time Match Generation UI

Match generation is asynchronous — it can take seconds to minutes. The challenge was keeping the UI responsive while polling for status updates without creating flicker or stale states. I built a polling system with 2-second intervals, optimistic session-based state to prevent UI jumps on page reloads, and a custom event emitter that notifies other parts of the app when generation completes. The result is a smooth experience where matches appear to stream in naturally.

Complex Filtering with Boolean Logic

Recruiters need filters far more powerful than simple dropdowns. The system supports boolean operators (AND/OR) across multiple filter groups, equality and range conditions, and saved filter presets. State management for this was tricky — filters affect GraphQL queries, table rendering, and URL state simultaneously. Redux Toolkit with carefully structured slices keeps everything in sync without performance issues on large datasets.

GraphQL State Management at Scale

Apollo Client's cache is powerful but can become unpredictable when mutations affect multiple queries. After a match decision (accept/reject), the cache needs to reflect the change across the matches table, the quick view, the best-match indicator, and the match count chips — all without a full refetch. I used targeted cache updates with writeQuery and carefully scoped refetchQueries to keep the UI consistent while minimizing network requests.

Result

In Production

Career Connect is actively used by recruiters to discover and manage candidate matches across open positions. The AI matching system processes real job requirements against real candidate profiles, with human feedback continuously improving match quality.

The frontend handles complex data-heavy workflows — advanced filtering, real-time generation polling, rich text editing, and multi-view navigation — while staying fast and accessible. Built with Next.js 14, TypeScript, GraphQL, and a component library of 45+ Radix UI primitives.
